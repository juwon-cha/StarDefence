# ⭐ Star Defence ⭐

## 1. 프로젝트 개요

**Star Defence**는 정해진 길을 따라 몰려오는 적들을 막기 위해 영웅을 배치하고 성장시켜 나가는 타워 디펜스 게임입니다. 플레이어는 재화를 사용하여 영웅을 소환 및 승급시키고 파괴된 타일을 수리하거나 특수 타일의 효과를 활용하는 등 다양한 전략적 선택을 통해 승리해야 합니다.

## 2. 시작하기

- **Unity Version:** `2022.3.62f3`
- **실행 방법:**
  1. 이 저장소를 복제(clone)합니다.
  2. Unity Hub에서 `2022.3.62f3` 버전의 Unity 에디터로 프로젝트를 엽니다.
  3. `Assets/Scenes/InGame.unity` 씬을 실행합니다.

## 3. 핵심 기능

- **영웅 소환 및 합성:** 골드를 사용해 무작위 영웅을 소환하고, 같은 등급의 영웅 둘을 합성하여 상위 등급 영웅으로 승급시킬 수 있습니다.
- **영구 강화 시스템:** 골드와 미네랄을 투자하여 영웅/지휘관의 능력치나 상위 등급 영웅 소환 확률을 영구적으로 높일 수 있습니다.
- **초월 시스템:** 최고 등급(Tier 3) 영웅을 초월시켜 고유한 신화 스킬을 가진 강력한 신화 등급으로 만들 수 있습니다.
- **자원 관리:** 탐사정을 구매하여 미네랄을 자동으로 채취하고 이를 타일 수리나 영웅 승급 등 전략적 요소에 사용합니다.
- **현상금 시스템:** 원할 때 강력한 적을 소환하여 처치하고 높은 보상을 얻는 "하이 리스크, 하이 리턴" 시스템입니다.
- **전략적 타일:** 맵에 무작위로 생성되는 강화 타일은 해당 위치에 배치된 영웅에게 공격 속도 증가와 같은 이로운 효과를 부여합니다.

## 4. 게임 플레이 루프

1.  **준비 단계 (`Build`):** 웨이브가 시작되기 전 플레이어는 타일을 클릭하여 영웅을 배치하거나 이미 배치된 영웅을 업그레이드합니다.
2.  **웨이브 시작 (`Wave`):** 웨이브 경고 버튼을 누르거나 시간이 다 되면 웨이브가 시작되고 적들이 경로를 따라 진격합니다.
3.  **전투:** 배치된 영웅들은 사거리 내의 적을 자동으로 공격합니다. 적을 처치하면 골드를 획득합니다.
4.  **자원 활용:** 전투 중에 얻은 골드와 탐사정이 채취한 미네랄을 사용하여 영웅을 추가로 소환/승급하거나 파괴된 타일을 수리하고 영구 강화를 진행할 수 있습니다.
5.  **웨이브 종료:** 웨이브의 모든 적을 성공적으로 막아내면 다시 준비 단계로 돌아가 다음 웨이브를 준비합니다.
6.  **승리 또는 패배:** 마지막 웨이브까지 모두 막으면 승리합니다. 만약 적이 최종 목적지에 도달하여 지휘관의 체력이 0이 되면 패배합니다.

## 5. 핵심 시스템 상세

### 5.1. 관리자 기반 아키텍처
게임의 핵심 로직은 각 기능을 전담하는 싱글톤 매니저 클래스들에 의해 관리됩니다.

- **`GameManager.cs`**: 게임의 전반적인 상태(Ready, Build, Wave, GameOver)와 재화(골드, 미네랄), 영웅 배치/승급/수리 등 핵심 로직을 총괄합니다.
- **`WaveManager.cs`**: `WaveDataSO`에 기반하여 적 웨이브를 생성하고 게임의 승리 조건을 관리합니다.
- **`GridManager.cs`**: 게임 맵의 타일(`Tile.cs`) 그리드를 생성 및 관리하고 강화 타일 생성 및 길찾기 데이터를 제공합니다.
- **`UpgradeManager.cs`**: 골드/미네랄을 사용한 영구 강화 및 초월 시스템을 담당합니다.
- **`BountyManager.cs`**: 현상금 몬스터 소환 및 보상 로직을 관리합니다.
- **`ProbeManager.cs`**: 탐사정의 구매 및 미네랄 채취 로직을 담당합니다.
- **`UIManager.cs`**: 게임 내 모든 UI(팝업, 버튼, 정보 표시)를 관리합니다.

### 5.2. 데이터 기반 설계(ScriptableObject)
영웅, 적, 웨이브, 강화 등 게임의 핵심 데이터는 `ScriptableObject`로 관리되고 코드를 직접 수정하지 않고도 데이터 수정 및 밸런스 조정이 용이합니다.

- **`CreatureDataSO`**: 영웅(`HeroDataSO`), 적(`EnemyDataSO`), 지휘관(`CommanderDataSO`) 등 모든 유닛의 기본 능력치(체력, 공격력 등)를 정의합니다.
- **`UpgradeDataSO`**: 각 강화 단계별 비용, 재화 종류, 효과 등을 정의합니다.
- **`WaveDataSO`**: 각 웨이브에 등장하는 적의 종류와 수를 정의합니다.
- **`BountyDataSO`**: 현상금으로 소환될 적과 보상 정보를 정의합니다.
- **`MythicSkillSO`**: 초월 영웅이 사용하는 신화 스킬의 구체적인 동작을 정의합니다.

## 6. 프로젝트 구조

`Assets` 폴더는 게임의 모든 리소스와 코드를 담고 있으며 기능별로 다음과 같이 구성되어 있습니다.

```
Assets/
├── 📂 Artwork/              # 게임에 사용되는 2D 이미지 리소스 (UI, 배경 등)
├── 📂 Prefabs/              # 게임 오브젝트의 원본 (프리팹)
│   ├── Creatures/         # 영웅, 적, 지휘관, 탐사정 프리팹
│   ├── Projectiles/       # 투사체 프리팹
│   └── UI/                # UI 요소 및 팝업창 프리팹
├── 📂 Resources/            # Resources.Load()로 동적 로드되는 애셋 (맵 데이터 등)
├── 📂 Scenes/               # 게임 씬 (.unity 파일)
├── 📂 ScriptableObjects/    # ScriptableObject 애셋 데이터
│   ├── Bounty/
│   ├── Buffs/
│   ├── Commander/
│   ├── Enemy/
│   ├── Hero/
│   ├── Probe/
│   ├── SKill/             # (Mythic Skills)
│   ├── Upgrade/
│   └── Wave/
└── 📂 Scripts/              # 모든 C# 스크립트 소스 코드
    ├── Assets/
    ├── Common/            # Singleton, Constants 등 공용 스크립트
    ├── Creatures/         # 영웅, 적 등 게임 유닛의 로직 (Base, Commander, Enemies, Heroes, Probes)
    ├── Data/              # 데이터 구조 관련 스크립트 (SO 정의)
    ├── Managers/          # 게임의 핵심 동작을 관리하는 모든 매니저 클래스
    ├── Stages/            # Tile, WaveDataSO 등 스테이지 구성 로직
    └── UI/                # UI 상호작용 및 표시 관련 로직
```
